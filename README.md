# 30 Days of JavaScript Projects 

A self-paced challenge to build **30 small JavaScript projects in 30 days** using only vanilla HTML, CSS, and JavaScript — no frameworks, no shortcuts.

By Day 30, you'll have a solid portfolio of real, working projects that demonstrate your frontend fundamentals.

---

##  Goals

- Master core JavaScript concepts through hands-on building
- Build a GitHub portfolio of 30 live projects
- Progress from DOM basics to APIs, storage, and mini-apps
- Develop consistency and daily coding discipline

---

##  Rules

1. **No frameworks** — Pure HTML, CSS, and JavaScript only
2. **One repo per project** — Each project gets its own GitHub repository
3. **Use localStorage** where data should persist between sessions
4. **Make it mobile-responsive** — Every project must work on small screens

##  Repository Structure

Each day has its own repository following this naming convention:

```
day-01-todo-list/
day-02-counter-app/
day-03-digital-clock/
...
day-30-portfolio-landing-page/
```

Each repo contains:

```
day-XX-project-name/
├── index.html
├── style.css
├── script.js
└── README.md
```

---

##  How to Run Any Project

1. Clone the specific day's repository:
   ```bash
   git clone https://github.com/Fok2002/day-XX-project-name.git
   ```
2. Open `index.html` in your browser — no build step needed.
